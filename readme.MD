
# ZPUB_READER // Sovereign Vault v1.2

A high-leverage digital asset ledger and EPUB reader designed for local-first deployment on Ubuntu Server. Optimized for the HP Elitebook architecture.

---

## 1. System Overview
The ZPUB_READER is a private infrastructure for the management and consumption of digital manuscripts. It operates on a "Sandbox" model, serving files through a static perimeter while maintaining a relational ledger via SQLite.

- **Backend:** Node.js / Express
- **Database:** SQLite3 (The Relational Ledger)
- **UI:** Japanese Minimalist / SQLite.org High-Density CSS
- **Host:** HP Elitebook (Ubuntu 22.04+)

3. Protocol: Installation & Deployment

3.1 Environment Setup

Ensure Node.js and SQLite3 are installed on the Ubuntu host. Initialize the
backend dependencies:

cd backend
npm install express sqlite3 dotenv

3.2 Perimeter Configuration

Configure the .env file in the /backend directory:

PORT=3000
DATABASE_URL=./vault.db

3.3 Execution

Invoke the server logic. Standard streams will log activity for auditing.

node server.js

5. Underwriting & Loss Prevention

  - Pathing Integrity: All HTML paths are absolute (starting with /).
  - Graceful Failure: The server implements a 404/500 loss prevention strategy,
    logging all unauthorized perimeter breaches to stdout.
  - Portability: The database is a single .db file. The entire system can be
    compressed and moved across Ubuntu nodes without data drift.
